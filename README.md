# AED1 UFG 2026

Um repositório acadêmico de algoritmos e estruturas de dados para aprendizado e pesquisa.

## Descrição do Projeto

Este repositório reúne implementações em C de exercícios acadêmicos sobre algoritmos e estruturas de dados. O projeto investiga o comportamento de soluções básicas e intermediárias em recursividade, busca, ordenação, matrizes, filas, pilhas e listas. Seu objetivo é servir como referência prática para estudantes de Análise e Desenvolvimento de Sistemas, Ciência da Computação e Engenharia da Computação.

## Contexto Acadêmico

Desenvolvido no âmbito da Universidade Federal de Goiás (UFG), sob orientação do professor Gustavo Machado Leal. O código foi elaborado como complemento de estudo e prática para disciplinas de algoritmos e estruturas de dados da UFG.

## Instruções de Instalação e Execução

### Requisitos

- Sistema operacional: Linux, Windows ou macOS
- Compilador C compatível com o padrão C99 ou superior (ex: `gcc`)

### Passo a passo

1. Clone o repositório:

```bash
git clone https://github.com/renatoibressan/aed1-ufg-2026-01.git
cd aed1-ufg-2026-01
```

2. Entre na pasta da lista desejada. Exemplo:

```bash
cd "Listas de Exercícios/Lista 02 - Recursividade/"
```

3. Compile o arquivo de interesse com `gcc`. Exemplo:

```bash
gcc R1.c -o R1
```

4. Execute o programa compilado:

```bash
./R1
```

### Observação

Cada subpasta contém um arquivo `main` ou arquivos de exercícios específicos para compilar e executar. Ajuste o nome do arquivo de entrada conforme o programa que deseja testar.

## Estrutura de Dados / Arquivos

O repositório está organizado por listas de exercícios. A estrutura principal é:

- `Listas de Exercícios/Lista 01 - Matrizes/`
  - `Mat1.c`, `Mat2.c`, `Mat3.c`
  - `main`
- `Listas de Exercícios/Lista 02 - Recursividade/`
  - `R1.c`, `R2.c`, `R3.c`, `R4.c`, `R5.c`
  - `main`
- `Listas de Exercícios/Lista 03 - Busca e Ordenação/`
  - `B1.c`, `B2.c`, `B3.c`, `B4.c`
  - `O1.c`, `O2.c`, `O3.c`
  - `main`
- `Listas de Exercícios/Lista 04 - Filas, Pilhas e Listas/`
  - `F1.c`, `F2.c`, `F3.c`, `F4.c`
  - `L1.c`, `L2.c`, `L3.c`
  - `P1.c`, `P2.c`, `P3.c`
  - `main`

Cada pasta contém implementações específicas de algoritmos ou estruturas, como matrizes e operações em pilhas, filas e listas encadeadas.

## Resultados e Casos de Uso

O código gera saídas de terminal que mostram o comportamento dos algoritmos em casos testes simples. Alguns exemplos de resultados possíveis:

- Validação de operações de matrizes e suas transformações.
- Impressão de sequências recursivas ou profundidade de chamadas recursivas.
- Comparação de buscas sequenciais e ordenações básicas.
- Simulação de inserção e remoção em filas, pilhas e listas.

### Como interpretar os resultados

1. Observe os valores de entrada e saída exibidos no terminal.
2. Compare os resultados com a lógica esperada do algoritmo estudado.
3. Utilize os exemplos de execução para validar conceitos em sala de aula ou trabalhos práticos.
